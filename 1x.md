# 一、Hearth 要开发哪些模块

我先给你一个总览：

```text
Hearth
├─ Core System
├─ Reticulum Runtime
├─ Interface Management
├─ Discovery & Routing
├─ Health / Watchdog
├─ Observability
├─ Security / Access Control
├─ Configuration Management
├─ Backup / Restore
├─ Plugin System
├─ Service Host
├─ Fleet / Central Management
├─ Topology / Network Intelligence
├─ Packaging / Deployment
├─ API Layer
├─ Web UI Layer
└─ CLI Layer
```

下面展开。

---

## 1. Core System（核心系统模块）

这是 Hearth 的系统骨架。

### 需要的模块

* **App Bootstrap**

  * 应用启动入口
  * 初始化配置、数据库、日志、依赖
* **Lifecycle Manager**

  * 整个 Hearth 的启动、停止、重启流程
* **Event Bus**

  * 内部事件分发
  * 例如 interface.up、runtime.down、peer.seen
* **Task Scheduler**

  * 周期任务
  * 健康检查、状态采样、清理历史数据
* **State Coordinator**

  * 汇总系统当前状态
  * 给 Dashboard/API 提供统一快照
* **Job Runner**

  * 执行异步任务
  * 备份、恢复、批量操作、升级等

### 作用

这一层是所有模块的宿主。
没有它，后面的模块都会散。

---

## 2. Reticulum Runtime（Reticulum 运行时模块）

这是 Hearth 的核心中的核心。

### 需要的模块

* **Runtime Adapter**

  * 对接 Reticulum
  * 启动/停止/重启 runtime
* **Runtime Supervisor**

  * 检查 runtime 是否正常
  * 发现异常并触发恢复
* **Identity Manager**

  * 节点 identity 初始化、保存、导入、导出
* **Runtime Status Collector**

  * 采集运行时状态
  * uptime、status、errors
* **Runtime Config Bridge**

  * Hearth 配置与 Reticulum 配置之间的映射
* **Degradation Policy Engine**

  * 节点退化运行策略
  * 例如某接口坏了不至于整机崩

### 作用

这一层负责把 Reticulum 从“一个底层运行对象”变成“一个可管理节点”。

---

## 3. Interface Management（接口管理模块）

这是 Hearth 的第二核心。

### 需要的模块

* **Interface Registry**

  * 注册所有接口类型
* **Interface Loader**

  * 按配置实例化接口
* **Interface Driver Base**

  * 抽象基类
* **TCP Interface Driver**
* **Serial Interface Driver**
* **RNode Interface Driver**
* **Local Network Interface Driver**
* **Custom Interface Driver Slot**

  * 给未来插件接口预留
* **Interface Status Monitor**

  * 接口在线/离线检测
* **Interface Metrics Collector**

  * RX/TX/errors/last_seen
* **Interface Restart Manager**

  * 单接口重启策略
* **Interface Validation**

  * 配置校验、设备存在性检查

### 作用

这一层让 Hearth 能真正管理：

* Internet uplink
* 局域网
* 串口设备
* RNode
* 未来的无线电桥接接口

---

## 4. Discovery & Routing（发现与路由模块）

这一层负责“看见网络”。

### 需要的模块

* **Peer Discovery Collector**

  * 记录最近见到的节点
* **Peer Repository**

  * peer 数据存储与查询
* **Route Snapshot Collector**

  * 抓取路径表快照
* **Route Repository**

  * 路由表数据存储与查询
* **Announce Collector**

  * 记录 announce
* **Announce Repository**

  * announce 查询
* **Path Learning Observer**

  * 观察路径学习变化
* **Network Snapshot Builder**

  * 形成当前网络状态快照
* **Peer Detail Resolver**

  * 聚合 peer 的来源、hop、最近活动

### 作用

这一层支撑：

* Peers 页面
* Routes 页面
* Announcements 页面
* Dashboard 上的网络摘要

---

## 5. Health / Watchdog（健康检查与自恢复模块）

这是 Hearth 的“基础设施感”来源。

### 需要的模块

* **Health Check Engine**

  * 节点健康检查
* **Health Rule Set**

  * 定义 healthy / warning / degraded / critical
* **Watchdog Manager**

  * 根据健康检查执行恢复动作
* **Recovery Action Executor**

  * 重启 runtime、重启 interface、切换模式
* **Restart History Recorder**

  * 记录自动恢复历史
* **Incident Tracker**

  * 异常事件跟踪
* **Maintenance Mode Manager**

  * 维护模式，避免自动恢复误触发
* **Health Score Calculator**

  * 节点健康评分

### 作用

保证节点是“长期在线可恢复”的，而不是“坏了你自己上 SSH 修”。

---

## 6. Observability（观测模块）

这部分不是重型监控平台，而是 Hearth 自身的可见性系统。

### 需要的模块

* **Metrics Aggregator**

  * 聚合节点、接口、网络指标
* **Timeline/Event Stream**

  * 事件时间线
* **Stats Summary Builder**

  * Dashboard 上的概览数据
* **Historical Sampler**

  * 轻量历史趋势采样
* **Prometheus Exporter**

  * `/metrics`
* **Alert Rule Hooks**

  * 将来给外部告警系统用
* **Log Aggregator**

  * 汇总应用日志、系统日志
* **Audit Log Store**

  * 用户操作审计

### 作用

支撑：

* Dashboard
* Logs
* System
* Health 状态卡片
* Prometheus 兼容输出

---

## 7. Security / Access Control（安全与访问控制模块）

1.x 必须有。

### 需要的模块

* **Auth Manager**

  * 登录/认证
* **Session Manager**

  * Web 会话
* **Token Manager**

  * API token、admin token
* **RBAC Manager**

  * 角色权限控制
* **Permission Evaluator**

  * 每个操作权限判断
* **User Manager**

  * 用户创建、禁用、角色变更
* **Audit Logger**

  * 谁执行了什么操作
* **Security Policy Manager**

  * LAN/WAN 访问策略
* **Secret Store**

  * token、敏感配置项管理

### 作用

解决：

* 谁能看
* 谁能改
* 谁能重启
* 谁能恢复备份
* 谁能管理插件/服务

这是 Hearth 的控制中枢之一。

### 需要的模块

* **Config Loader**

* **Config Schema Validator**

  * 保存变更

  * 配置版本历史

  * 比较配置变化
* **Config Apply Manager**

* **Template Renderer**

* **Environment Resolver**

  * 不同系统环境下路径/变量解析

### 作用

支撑：

* Configuration 页面
* Fleet 模板配置
* 安全地修改节点设置

## 9. Backup / Restore（备份恢复模块）

非常重要。

### 需要的模块

* **Backup Builder**

  * 打包配置、identity、数据库

  * 记录版本、时间、主机信息
* **Restore Validator**

  * 恢复前校验
* **Restore Executor**

  * 执行恢复
* **Snapshot Manager**

  * 节点快照
* **Export/Import Service**

  * 导出导入
* **Backup Retention Manager**

  * 自动保留策略
* **Disaster Recovery Helper**

  * 恢复流程辅助

### 作用

让 Hearth 真正可迁移、可恢复，而不是一次性节点。

## 30. Plugin Sources

显示：

* 插件源/索引
* 是否信任
* 同步状态
* 签名状态
* 签名算法
* 公钥
* manifest SHA256
* 同步错误

当前实现补充：

* 插件源清单已支持 **Ed25519 公钥签名校验**
* `public_key` + `signature_algorithm = "ed25519"` 是推荐配置方式
* `manifest_sha256` 保留为可观测字段，不再作为“真正签名”本体

推荐配置示例：

```toml
[[plugin_sources]]
name = "community"
index_url = "https://example.net/hearth/community.json"
label = "Community Source"
public_key = "ed25519:0123456789abcdef0123456789abcdef0123456789abcdef0123456789abcdef"
signature_algorithm = "ed25519"
signature_required = true
```
